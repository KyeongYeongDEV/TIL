## **1. 단기 / 중기 / 장기 스케줄러**

| 종류 | 설명 | 실행 시점 | 비유 |
| --- | --- | --- | --- |
| **장기 스케줄러 (Long-term)** | 디스크(잡 큐)에 있는 **작업** 중 어떤 걸 메모리(Ready Queue)로 올릴지 결정 | 작업이 처음 시스템에 들어올 때 | 공장에 어떤 원자재를 반입할지 정하는 담당 |
| **중기 스케줄러 (Medium-term)** | 메모리에 있는 프로세스를 **일시적으로 제거(Suspend)** 하거나 다시 불러와서 부하 조절 | 메모리 부족·부하 분산 필요할 때 | 작업 중인 기계를 잠깐 멈추고 다른 작업기로 전환 |
| **단기 스케줄러 (Short-term)** | Ready Queue에 있는 프로세스 중 **CPU를 누구에게 줄지** 결정 | CPU가 놀 때마다 | 당장 기계를 쓸 작업자를 뽑는 관리자 |

---

## **2. 현대 OS에서 세 가지를 모두 쓰나?**

- **현대 OS 대부분**은 **단기 스케줄러**는 반드시 사용
- **장기 스케줄러**는 **서버/메인프레임** 등 배치 작업 환경에서 주로 사용하고, **일반 PC·모바일 OS는 거의 안 씀**
- **중기 스케줄러**는 **스와핑(Swapping)**, 메모리 압박 시 프로세스 suspend 같은 기능에서 부분적으로만 사용됨

> 즉, 오늘날 PC/스마트폰 OS = 단기 + (필요 시 중기)
> 
> 
> 장기는 잘 안 씀.
> 

---

## **3. 프로세스 스케줄링 상태**

1. **New** – 생성 중
2. **Ready** – CPU 차례 기다림
3. **Running** – CPU 사용 중
4. **Waiting/Blocked** – I/O 요청 등으로 대기
5. **Terminated** – 실행 끝
6. **Suspended** – 메모리에서 내려가 디스크에 잠시 대기 (중기 스케줄러 개입 시)

---

## **4. Preemptive / Non-preemptive에서 존재할 수 없는 상태?**

- **모든 상태는 둘 다 존재 가능**
- 차이는 **CPU를 빼앗길 수 있느냐 없느냐**
    - Preemptive: 타이머나 우선순위로 CPU 강탈 가능 → Running → Ready 전환 가능
    - Non-preemptive: CPU 스스로 양보할 때만 전환됨 → Running → Ready 전환 **없음**

---

## **5. Memory 부족 시 프로세스 상태 변화**

- 메모리 부족 시 **중기 스케줄러**가 개입
- 실행 중/대기 중 프로세스를 **Suspended 상태**로 보내고, 디스크(스왑 영역)에 저장
- 메모리 여유 생기면 다시 Ready 상태로 복귀

> 흐름:
> 
> 
> `Running / Waiting → Suspended`
> 
> `Suspended → Ready` (복귀 시)
> 

---

💡 **면접 답변 예시 한 줄 요약**

> "장기는 작업을 메모리에 들이는 역할, 중기는 메모리 부족 시 내보내고, 단기는 CPU 줄 사람을 고릅니다. 현대 OS는 단기 위주, 필요 시 중기 사용, 장기는 거의 안 씁니다. 프로세스는 New, Ready, Running, Waiting, Terminated, Suspended 상태를 가질 수 있고, Preemptive에서는 Running에서 Ready로 강제 전환이 가능하지만 Non-preemptive에서는 없습니다. 메모리가 부족하면 프로세스는 Suspended로 이동했다가 복귀합니다."
> 

# 면접 대답 정리

**질문:** 단기, 중기, 장기 스케줄러에 대해 설명해 주세요.

**대답:**

> 운영체제에서 스케줄러는 CPU나 메모리 같은 자원을 어떤 프로세스에 줄지 결정하는 역할을 합니다.
> 
> 
> 장기 스케줄러는 디스크에 있는 작업 중 어떤 걸 메모리에 올릴지, 중기 스케줄러는 메모리가 부족하거나 부하를 줄이기 위해 프로세스를 잠시 디스크로 내렸다 올리는 역할을 합니다.
> 
> 단기 스케줄러는 CPU를 당장 누구한테 줄지를 결정하는 역할이고, 현대 OS에서는 주로 단기를 쓰고, 중기는 일부 상황에서, 장기는 거의 사용하지 않습니다.
> 

---

**질문:** 프로세스의 스케줄링 상태에 대해 설명해 주세요.

**대답:**

> 프로세스는 생성(New), 준비(Ready), 실행(Running), 대기(Waiting), 종료(Terminated), 그리고 메모리에서 내려간 일시중단(Suspended) 상태가 있습니다.
> 
> 
> Ready는 CPU를 기다리는 상태, Waiting은 I/O를 기다리는 상태, Suspended는 중기 스케줄러가 개입했을 때 생깁니다.
> 

---

**질문:** preemptive / non-preemptive에서 존재할 수 없는 상태가 있을까요?

**대답:**

> 상태 자체는 다 존재하지만, Preemptive에서는 실행 중(Running)인 프로세스를 강제로 Ready로 돌릴 수 있고, Non-preemptive에서는 이런 전환이 없습니다.
> 

---

**질문:** Memory가 부족할 경우, Process는 어떠한 상태로 변화할까요?

**대답:**

> 메모리가 부족하면 중기 스케줄러가 실행 중이거나 대기 중인 프로세스를 Suspended 상태로 바꾸고 디스크에 보관합니다.
> 
> 
> 나중에 메모리가 여유 생기면 다시 Ready 상태로 복귀시킵니다.
>
